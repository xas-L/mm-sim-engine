# MarketSim v2.1 : Market Making Simulator (C++ & Streamlit)

A compact MM sandbox built to learn the *full loop* end-to-end:  
**limit order book → external flow → MM quoting policy → fills → accounting → telemetry/trade tape → diagnostics report**.

This is **not** a high-fidelity exchange emulator. It’s an **explainable simulation** designed to stress the right failure modes (inventory risk, adverse selection, regime bursts) while producing clean outputs you can sanity-check.

---

## "What's all this then?"

Each run produces **three artifacts**:

1. **Telemetry CSV** : one row per step (end-of-step snapshot)
   - state: time/step, bid/ask, my quotes, inventory  
   - P&L: cash, realised/unrealised, equity  
   - operations: fills/cancels (cumulative counters)  
   - microstructure: spread, imbalance, microprice, time-at-touch

2. **Trades CSV** :  one row per print
   - includes whether **my MM participated**, plus queue diagnostics (maker age, queue ahead)

3. **PDF summary report** (generated by Streamlit)
   - equity & drawdown, inventory, top-of-book, P&L distribution, rolling Sharpe (diagnostic only), markout-style plots

---

## Why this project exists

I built this to learn market making by implementing the mechanics :
- matching rules & tick grid
- queue/FIFO intuition
- how inventory creates drawdowns
- why short-horizon markout can be negative even if longer-horizon is positive
- why sanity checks (accounting + inventory reconstruction) prevent self-deception

---

## Architecture (high level)

### C++ sim 
- Tick grid handling (prices snapped to a tick)
- LOB with price levels + FIFO queues
- External flow model:
  - limit replenishment + market orders
  - simple regime switching (normal vs toxic bursts)
  - occasional sweeps to create gap risk
- Market-making agent:
  - quotes around a mark (`mid_mark`)
  - inventory skew + size control
  - cancel latency / minimum quote life
  - emergency flattening when inventory too large
- Accounting:
  - logs telemetry after matching + fees
  - key identity (should hold each step):  
    `equity = cash + inv_value`

### Python/Streamlit viewer
- Loads telemetry + trades
- Computes derived series (Δequity, drawdown, rolling metrics)
- Flags implausible outputs (e.g. time-mapping mistakes on Sharpe)
- Exports a standard PDF report for review / sharing
